# TerrainScanner - Система корреляционного анализа рельефа

## Описание проекта

**TerrainScanner** - это демонстрационное приложение для симуляции полета дрона и анализа корреляции между картографическими данными рельефа и показаниями радиовысотомера.

### Возможности:
- Загрузка карт высот в формате GeoTIFF
- Симуляция полета дрона по заданным параметрам
- Поддержка внешних данных NMEA ($GPGGA)
- Визуализация маршрута полета на карте
- Построение графиков корреляции
- Расчет статистических метрик (Pearson, Spearman, RMSE, MAE, R²)

## Установка и запуск

### Способ 1: Из исходного кода

1. **Клонируйте репозиторий:**
```bash
git clone https://github.com/Glaz-almaz-GL/Hackathon-2606.git
cd TerrainScanner
```

2. **Восстановите зависимости:**
```bash
dotnet restore
```

3. **Соберите проект:**
```bash
dotnet build --configuration Release
```

4. **Запустите приложение:**
```bash
dotnet run --project TerrainScanner.WinForms
```

Или откройте файл `TerrainScanner.sln` в Visual Studio и нажмите F5.

### Способ 2: Готовый исполняемый файл

Если предоставлен готовый билд:
```bash
cd TerrainScanner.WinForms\bin\Release\net8.0-windows
.\TerrainScanner.WinForms.exe
```

## Инструкция по использованию

### Шаг 1: Загрузка карты высот

1. Запустите приложение
2. Нажмите кнопку **"Выбор карты"** в верхней панели
3. В открывшемся диалоге выберите файл карты в формате **.tif** (GeoTIFF)
   - Файл должен содержать данные высот (DTM/DEM)
   - Поддерживаются стандартные GeoTIFF файлы с геопривязкой

**Пример файлов карт:**
- Файлы SRTM (Shuttle Radar Topography Mission)
- Файлы ASTER GDEM
- Любые другие GeoTIFF с данными высот

### Шаг 2: Настройка параметров симуляции

1. Нажмите кнопку **"Симулировать полет дрона"**
2. В открывшемся окне введите параметры:

| Параметр | Описание | Пример |
|----------|----------|--------|
| **Шум радиовысотомера (м)** | Амплитуда шума измерений | 30.0 |
| **Скорость дрона (м/с)** | Скорость полета | 50 |
| **Время полета (с)** | Длительность симуляции | 60 |
| **Высота полета (м)** | Высота над уровнем моря | 2500 |
| **Азимут** | Курс полета (градусы) | 0.0 |
| **Широта** | Стартовая широта | 55.726389 |
| **Долгота** | Стартовая долгота | 109.385139 |
| **Частота получения данных (Гц)** | Частота замеров | 1.0 |

3. **Опционально**: 
   - Отметьте **"Использовать ячейки?"** для ввода координат в индексах сетки
   - Отметьте **"Использовать файл с замерами?"** для загрузки внешних данных

4. Нажмите **OK**

### Шаг 3: Использование внешних данных (опционально)

Если вы хотите использовать реальные данные радиовысотомера:

1. Подготовьте текстовый файл с NMEA-сообщениями формата GGA:
```
$GPGGA,120000.000,,,,,,0,00,,,1245.3,M,,M,,*00
$GPGGA,120001.000,,,,,,0,00,,,1243.8,M,,M,,*00
$GPGGA,120002.000,,,,,,0,00,,,1241.5,M,,M,,*00
```

2. В диалоге параметров отметьте **"Использовать файл с замерами?"**
3. Укажите путь к файлу

### Шаг 4: Анализ результатов

После запуска симуляции:

**Верхняя панель (Карта):**
- Отображается карта высот в оттенках серого
- Красная линия - маршрут полета дрона
- Зеленый квадрат - точка старта
- Желтый квадрат - текущая позиция

**Нижняя панель (График):**
- **Зеленая линия** - ожидаемые данные высот из карты
- **Оранжевая линия** - измеренные данные (с шумом или из файла)
- Чем ближе линии друг к другу, тем выше точность

**Левая панель (Статистика):**
- **Коэффициент корреляции Пирсона** - линейная корреляция (0-1, чем ближе к 1, тем лучше)
- **Коэффициент Спирмена** - ранговая корреляция
- **RMSE** - среднеквадратичная ошибка (в метрах, чем меньше, тем лучше)
- **MAE** - средняя абсолютная ошибка (в метрах)
- **R²** - коэффициент детерминации
- **Ширина/Высота клетки** - размер пикселя карты в метрах

**Нижняя панель (Лог):**
- Отображает процесс выполнения
- Сообщения об ошибках
- Статистику симуляции

## Ожидаемые результаты

### Успешная симуляция:

1. **На карте** должен отобразиться путь дрона (красная линия)
2. **На графике** должны появиться две линии:
   - Зеленая (карта) - плавная, без шума
   - Оранжевая (высотомер) - с небольшими отклонениями
3. **В статистике**:
   - Pearson: > 0.95 (отлично), 0.8-0.95 (хорошо), < 0.8 (требует проверки)
   - RMSE: должно быть близко к заданному уровню шума

### Пример хороших результатов:
```
Коэффициент корреляции Пирсона: 0.9993
Коэффициент ранговой корреляции Спирмена: 0.9975
Среднеквадратичная ошибка: 17.21 м
Средняя абсолютная ошибка: 14.92 м
Коэффициент детерминации: 0.9942
```

## Устранение неполадок

### Проблема: "Карта не загружена"
**Решение**: Убедитесь, что выбран файл .tif перед запуском симуляции

### Проблема: "Симуляция не вернула ни одной точки"
**Возможные причины**:
- Стартовая точка за пределами карты
- Начальная ячейка содержит NoData (NaN)
- Скорость/длительность слишком малы

**Решение**: 
- Проверьте координаты стартовой точки
- Увеличьте время полета или скорость
- Выберите другую стартовую позицию

### Проблема: UI зависает при большой карте
**Решение**: 
- Уменьшите частоту получения данных (Гц)
- Уменьшите время полета
- Используйте карты меньшего разрешения

### Проблема: Низкая корреляция
**Возможные причины**:
- Слишком высокий уровень шума
- Ошибки в данных NMEA
- Дрон вышел за границы карты

**Решение**:
- Уменьшите параметр "Шум радиовысотомера"
- Проверьте формат NMEA-файла
- Убедитесь, что маршрут не выходит за границы

## Форматы файлов

### GeoTIFF (.tif)
Стандартный формат растровых данных с геопривязкой. Должен содержать:
- Данные высот (один канал)
- GeoTransform (аффинное преобразование)
- Систему координат (желательно WGS84)

### NMEA (.txt)
Текстовый файл с сообщениями GGA:
```
$GPGGA,<время>,<широта>,<N/S>,<долгота>,<E/W>,<качество>,<спутники>,<HDOP>,<высота>,M,...
```

## Дополнительные возможности

### Горячие клавиши
- **Ctrl+O** - Открыть карту
- **Ctrl+S** - Сохранить результаты (в разработке)
- **Esc** - Отменить симуляцию

### Логирование
Все события записываются в лог (нижняя панель):
- Загрузка карты
- Параметры симуляции
- Ошибки и предупреждения
- Результаты расчетов

Лог можно очистить кнопкой **"Очистить лог"**
